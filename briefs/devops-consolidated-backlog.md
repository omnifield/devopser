## §0. Ревизия — что изменилось против v1 (и твоего триажа)

v1 этого файла содержала две ошибки оракул-архитектора, обе поймал user:

1. **Отсылки «смотри в capsule/…» — недопустимы.** Бриф = самодостаточная точечная спека;
   исполнитель не лазит в чужой/старый репо. Референсы допустимы только на конкретные файлы
   живых v2-репо.
2. **«Перенос стеков из оракула» отменён как класс работ.** Grafana/OTEL-обвязка агентских
   сессий, minio, preview — капсульные импровизации. Агент-наблюдаемость в v2 = **продукт
   brainer** (пульт — его интерфейс, не Grafana): если brainer'у нужна телеметрия-инфра,
   его архитектор закажет её сам, точечно, под свой контракт. То же с любым стеком: он
   появляется у тебя ТОЛЬКО под заказ конкретного потребителя.

По твоему триажу v1: **амендмент registry-npm в infra-migration — реверсни** (registry
переезжает пунктом скелет-трека, §1.3 ниже; сам infra-migration снимается — §2);
**preview-server — вопрос снят** (паркуем; твоя рекомендация совпала с решением);
workstation — без изменений.

## §1. Актуальный бэклог (приоритет сверху вниз)

1. **Workstation e2e (DoD)** — как было (`workstation/escalation-bootstrap-gaps.md`), ждёт
   среду от user.
2. **Repo-skeleton** (`briefs/repo-skeleton-product.md`, порядок D1→D3→D2, D4 после):
   спека там самодостаточна; живой референс D1 — weber
   `.github/workflows/ci.yml` + `pr-title.yml` (CI зелёный; sherif-шаг + permissions-фикс).
3. **registry-npm — зависимость D2** (не «миграция»): self-hosted npm-registry (Verdaccio
   дефолт; иное — предложи с обоснованием). Точечная спека: поднимается compose'ом;
   persistent storage; доступ с workstation (localhost); порт фиксируется в
   `registry/ports.md`; флоу publish/install для пресет-пакетов D2 (и будущих публикаций
   экосистемы). DoD: пакет publish→install с чистой машины по README.
4. **D4-оценки** (devcontainer / Renovate self-hosted / mise / self-hosted nx remote-cache) —
   по мере свободы, отдельными решениями.
