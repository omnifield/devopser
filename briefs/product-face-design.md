## Что такое лицо (просто)
**У каждого продукта есть описание «что и зачем». Лицо = шаблонная страница с этим описанием, на
роуте продукта.** Не сущность, не контейнер, не «вид лица». Один шаблон на все продукты.

- **Источник текста** — README продукта (у всех есть; у weber — «что и зачем» уже расписано).
  Шаблон-лицо его показывает. Ничего нового писать не надо.
- **Один шаблон** (часть продукт-скелета) — приезжает всем продуктам, как остальной managed-набор.
- **Отдаётся статикой** (хаб/gateway), **без контейнера на продукт**.

## Как работает
1. **Скелет** даёт продукту шаблон-лицо (рендерит его README/описание в страницу).
2. **Хаб** показывает лицо на reach-роуте продукта: `/weber` → описание weber.
3. Пакетный продукт (weber) — лицо и есть его присутствие в хабе (юзер-интерфейса нет).

## DoD
- `:8080/weber` показывает описание weber (что за фреймворк, пакеты `@weber/*`, зачем) — из README,
  по общему шаблону.
- Шаблон-лицо в скелете → приезжает любому продукту синком.
- Ни строки лица руками, ни отдельного контейнера.

## Открытый нюанс (не форк, уточним при сборке)
Интерактивные продукты (brainer) имеют ещё и **своё приложение** (пульт). Лицо-описание и приложение —
разное: описание = «что и зачем» (общий шаблон), приложение = сам UI (капабилити Develop/Agents/open).
Как их разложить по роутам (brainer `/brainer` = приложение, а описание — на карточке/подроуте?) —
мелочь, решим на сборке, модель не трогает.

## Границы
- Лицо = только описание (что/зачем), из README. Не рендерит логику продукта.
- Sandbox-app'ы weber — внутренние тест-полигоны, в хаб не выносятся.
