# Brief-заказ — owner-registry: порты продуктов → внутренние (single-origin)

| | |
|---|---|

| **От** | devopser-архитектор, 2026-07-12 |
| **Основание** | `gateway-network-single-origin.md` gw-step5. Gateway переехал на docker-сеть (апстримы по alias'у, не host.docker.internal) → порты продуктов больше НЕ хост-ресурс |
| **Зона** | только `registry/` — ничего вне |

## Задача

1. **`registry/ports.md`** — довести forward-note (я уже вписал предупреждение) до факта:
   - Порты продуктов (3500/5173/8010/8020/…) = **внутренние порты контейнеров** (docker-сеть
     `omnifield-gateway`), НЕ хост-порты. Убрать рамку «единый реестр портов dev-**хоста**» /
     «коллизии на хосте» — она держалась только на per-service host-publish, которого больше нет.
   - **`:8080` — единственный хост-контракт** (gateway). 9443 (Portainer, localhost) остаётся как есть.
   - Продуктовая таблица: порт = внутренний upstream + docker-alias (= имя репо), по которому его
     достаёт gateway. Глобальная уникальность портов между продуктами больше НЕ требуется (разные
     контейнеры); уникальность — в рамках одного devbox (два сервиса в одном контейнере — brainer
     3500+8010 — обязаны различаться).
   - Оставить пометку: капсульные/оракульские порты (§«Занято оракулом») — легаси-эпоха, к разбору
     не тащить.

2. **`registry/products.md`** — сверить, что каталог согласован (alias = имя репо = join-key
   `reach.routes[].service`, `liaison-inc1-manifest-boundary.md`). Файл уже сидовый — только
   поддержать актуальность, не переписывать.

## DoD
`registry/ports.md` не называет порты продуктов хостовыми; `:8080` явно единственный хост-контракт;
глобальная-уникальность-рамка снята. `products.md` согласован. Коммит в зоне `registry/`.

## Границы
- Только `registry/`. `nginx.conf`/`compose.yml`/devbox — НЕ твоё (сделано/handoff'ы в
  `gateway-network-single-origin.md`).
- Изменение самого ЗНАЧЕНИЯ порта (не лейбла «внутренний/хостовый») = контракт через architect —
  здесь только переклассификация, значения не трогаем.
